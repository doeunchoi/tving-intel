# TVING Contents Intelligence — 운영 가이드

정적 사이트 + CSV 데이터 구조입니다. **데이터 업데이트 = 리대시에서 받은 CSV를 `data/` 폴더에 덮어쓰고 재배포**가 전부입니다.
리대시 API 키·쿼리는 이 사이트에 일절 포함되지 않습니다 (사내망에서 사람이 CSV를 받아 옮기는 구조).

```
contents_intelligence/
├── index.html        ← 사이트 본체 (데이터 없음, data/*.csv를 읽어 렌더링)
├── preview.bat       ← 더블클릭 → 로컬 미리보기 서버 + 브라우저 자동 오픈
└── data/
    ├── lineup.csv    ← 01 탭: Hot & New 라인업 (편성중 + 공개예정)
    ├── rank.csv      ← 01 탭: 성·연령별 TOP 50
    ├── target.csv    ← 02·03 탭: 콘텐츠별 도달/집중도/브랜디드 지표
    └── meta.csv      ← 데이터 기준일 등 (헤더 우측 배지에 표시)
```

> ⚠️ 현재 `data/`의 CSV는 **더미데이터**입니다. 실데이터 리대시 쿼리가 준비되면 그대로 교체하세요.
> ⚠️ 이 사이트는 public 배포 전제입니다. **CSV에 담긴 데이터 자체가 외부에 공개**되므로, 절대 시청자 수 등 비공개 지표는 CSV에 넣지 마세요 (인덱스·구간 라벨 등 상대값만).

---

## 1. 데이터 업데이트 루틴 (주 1회 권장, 약 10분)

1. **리대시에서 CSV 다운로드** — 아래 4개 쿼리를 즐겨찾기해두고 실행 → Download as CSV.
   쿼리 내용을 수정하지 말 것 (컬럼 변경은 사이트 코드 수정과 세트로만, 아래 §3).
2. **파일 교체** — 다운로드한 파일을 정확한 이름(`lineup.csv` 등)으로 `data/`에 덮어쓰기.
3. **기준일 갱신** — `meta.csv`의 `updated_at` 값을 오늘 날짜로 수정.
4. **로컬 확인** — `preview.bat` 더블클릭 → 브라우저에서 3개 탭 모두 확인.
   - 데이터에 문제가 있으면 화면 상단에 **빨간 에러 박스로 원인(파일명·누락 컬럼)이 표시**됩니다.
   - 헤더 우측 "데이터 기준 YYYY-MM-DD" 배지가 갱신됐는지 확인.
5. **배포** — 호스팅 방식에 따라:
   - GitHub Pages: `git add data/ && git commit -m "data: 2026-06-10 갱신" && git push`
   - S3 등 정적 호스팅: `data/` 폴더만 업로드.
6. **배포 확인** — 실제 URL 접속, 기준일 배지 확인.

문제가 생기면: git 사용 시 직전 커밋으로 롤백(`git revert`), 아니면 이전 CSV 백업으로 복원.

## 2. 리대시 쿼리가 맞춰야 할 CSV 스키마

쿼리 SELECT 절에서 컬럼 alias를 아래와 **정확히 일치**시키면 다운로드 후 가공 없이 그대로 사용 가능합니다.
(대소문자 무관, 컬럼 순서 무관, 추가 컬럼은 무시됨. 다중 값 구분자는 `|`)

### lineup.csv
| 컬럼 | 설명 | 예시 |
|---|---|---|
| status | `onair`(편성중) / `soon`(공개예정) | onair |
| title | 콘텐츠명 | 환승연애4 |
| genre | 대표 장르 1개 | 예능 |
| sub_genres | 서브장르, `\|` 구분 | 연애\|리얼리티 |
| rating | `전체` / `12` / `15` / `19` | 15 |
| cast | 출연자, `\|` 구분 | 정현\|이미주 |
| premiere | 첫공개일 (표시용 문자열) | 2026/6/1 |
| days | 공개 요일 | 금 |
| episodes | 회차 정보 | 12부작 |
| is_original / is_tving_only / is_special | `1`/`0` (true/false도 허용) | 1 |
| poster_url | 포스터 이미지 URL, 비우면 장르 그라데이션 폴백 | https://… |

### rank.csv (모든 데모 조합을 한 파일에, long 포맷)
| 컬럼 | 설명 | 예시 |
|---|---|---|
| gender | `F` / `M` / `MF`(남녀 전체) | F |
| age | `10`~`60`, 또는 빈 값(전체 연령) | 20 |
| rank | 1~50 | 1 |
| week | 추출 주차 (화면 미표시, 데이터 이력용) | 6월 1주차 |
| title | 콘텐츠명 | 선재 업고 튀어 |
| main_genre | 메인 장르 | 드라마 |
| sub_genre | 서브 장르 (없으면 빈 값 가능) | 로맨스 |

**데모 세그먼트 매핑 규칙:**
- `F` / `10` → 10대 여성
- `M` / `30` → 30대 남성
- `MF` / `` → 남녀 전체 (전체 연령)
- `F` / `` → 여성 전체
- `M` / `` → 남성 전체
- `MF` / `20` → 20대 전체 (남녀)

### target.csv
| 컬럼 | 설명 | 예시 |
|---|---|---|
| title / genre | 콘텐츠명 / 장르 | 환승연애4 / 예능 |
| band | `소형`/`중형`/`대형`/`초대형` | 대형 |
| reach | 도달 게이지 0~100 | 74 |
| branded | 브랜디드 가능 여부 `1`/`0` | 1 |
| f10 f20 f30 f40 m10 m20 m30 m40 | 데모별 인덱스 (100=평균) | 186 |
| cast_info | `이름:역할:설명`을 `\|`로 연결 | 정현:MC:방송인\|이미주:패널:아이돌 |

### meta.csv (key,value 형식 — 수동 관리)
| key | 용도 |
|---|---|
| updated_at | 헤더 배지 "데이터 기준 YYYY-MM-DD" |
| rank_period | 랭킹 하단 문구 (예: 최근 4주 롤링) |

## 3. 스키마(컬럼) 변경 시

리대시 쿼리 컬럼을 바꾸면 `index.html`의 두 곳을 같이 수정해야 합니다:
- `initData()` 안의 `loadCSV(파일명, [필수 컬럼 목록])`
- 해당 매핑 함수 (`mapLineup` / `mapTarget` / rank 매핑부)

쿼리 변경과 코드 변경을 **같은 커밋으로 배포**하세요. 한쪽만 나가면 에러 박스가 뜹니다(의도된 안전장치).

## 4. 운영 원칙

- **CSV를 손으로 고치지 않는다.** 데이터가 틀리면 리대시 쿼리를 고치고 다시 다운로드. (예외: meta.csv)
- **리대시 키/쿼리 URL을 이 저장소·사이트에 절대 넣지 않는다.**
- git으로 관리 권장: 데이터 변경 이력 = 커밋 이력, 롤백 즉시 가능.
- 인코딩은 리대시 다운로드 원본(UTF-8) 그대로 사용. 엑셀로 열어 저장하면 인코딩이 깨질 수 있음.

## 5. 향후 반자동화 (선택)

리대시는 사내망 전용이므로 클라우드 CI(GitHub Actions 등)에서 직접 호출은 불가.
대신 **사내망 PC에서 도는 스크립트**(작업 스케줄러 등록)로:
리대시 API 호출(키는 PC에만 보관) → CSV 저장 → `git commit & push` → Pages 자동 재배포.
사이트 구조는 지금 그대로 두고 §1의 1~5단계만 스크립트로 대체하면 됩니다.
